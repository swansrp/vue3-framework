## Vue3 响应式系统循环依赖问题

### 问题描述
在 Vue3 中，当父子组件之间存在相互引用并且在 `computed` 中访问子组件的响应式数据时，容易触发 "Maximum recursive updates exceeded" 错误。

**典型场景**：
- 容器型组件需要根据子组件的状态计算自身进度
- 在 `computed` 中通过 ref 访问子组件并调用其方法获取状态
- 子组件状态变化 → 触发父组件 computed 重新计算 → 可能导致子组件重新渲染 → 循环依赖

### 错误案例
```typescript
// ❌ 错误：在 computed 中访问子组件的响应式数据
const childRefs = ref<Map<string, any>>(new Map())

const progress = computed(() => {
  const refs = childRefs.value  // 访问响应式 ref
  let completed = 0
  refs.forEach(childRef => {
    if (childRef.getProgress().percent === 100) {
      completed++
    }
  })
  return { completed, total: refs.size }
})
```

### 解决方案

#### 方案一：使用非响应式容器 + 手动更新（推荐）

**核心思路**：
1. 使用非响应式的 `Map` 存储子组件引用（避免触发响应式系统）
2. 使用 `ref` 存储计算结果作为缓存
3. 提供手动更新方法，在合适的时机调用

```typescript
// ✅ 正确：使用非响应式 Map
const childRefs = new Map<string, any>()  // 非响应式

// 使用 ref 缓存计算结果
const containerProgress = ref({ filled: 0, total: 0, percent: 0 })

// 手动更新进度的方法
const updateContainerProgress = () => {
  const refs = Array.from(childRefs.values())
  let completed = 0
  
  refs.forEach(childRef => {
    if (childRef?.getProgress()?.percent === 100) {
      completed++
    }
  })
  
  containerProgress.value = {
    filled: completed,
    total: refs.length,
    percent: refs.length > 0 ? Math.round(completed / refs.length * 100) : 0
  }
}

// computed 只返回缓存值，不触发计算
const progress = computed(() => {
  if (isContainerType.value) {
    return containerProgress.value  // 直接返回缓存
  }
  // 其他情况的计算逻辑
  return calculateProgress()
})

// 在合适的时机更新
onMounted(() => {
  nextTick(() => {
    updateContainerProgress()
  })
})
```

**设置子组件 ref 的正确方式**：
```typescript
// 返回稳定的函数引用
const setChildRef = (id: string) => {
  return (el: any) => {
    if (el) {
      childRefs.set(id, el)
    } else {
      childRefs.delete(id)
    }
  }
}

// 在模板中使用
// <ChildComponent :ref="setChildRef(child.id)" />
```

#### 方案二：事件驱动更新

适用于需要实时同步状态的场景：

```typescript
// 子组件暴露事件
const emit = defineEmits<{
  (e: 'progressChange', progress: { percent: number }): void
}>()

// 状态变化时触发事件
watch(() => progress.value, (newProgress) => {
  emit('progressChange', newProgress)
})

// 父组件监听事件
const childProgress = reactive(new Map<string, number>())

const handleChildProgressChange = (childId: string, progress: any) => {
  childProgress.set(childId, progress.percent)
  updateContainerProgress()
}
```

### 关键要点

1. **避免在 computed 中访问子组件**
   - computed 应该是纯函数，只依赖明确的响应式数据源
   - 避免在 computed 中调用子组件的方法

2. **使用非响应式容器存储引用**
   - 使用普通的 `Map` 或 `Object` 而不是 `ref` 或 `reactive`
   - 避免触发不必要的响应式追踪

3. **手动控制更新时机**
   - 在 `onMounted` + `nextTick` 中初始化
   - 在保存、提交等关键操作后手动更新
   - 暴露更新方法供外部调用

4. **避免每次渲染都创建新函数**
   - ref 回调函数应该返回稳定的引用
   - 使用闭包保持函数引用稳定

## Ant Design Tree 组件样式对齐问题

### 问题描述
当使用 `<a-tree>` 组件并自定义 `#title` 插槽时，容易出现图标与文本不对齐的问题。

### 解决方案

#### 1. 树节点容器使用 flex 布局
```less
.ant-tree-treenode {
  display: flex;
  align-items: center;  // 关键：让所有子元素垂直居中对齐
}
```

#### 2. 统一所有元素的行高
```less
// 内容包装器
.ant-tree-node-content-wrapper {
  line-height: 22px;
  vertical-align: middle;
}

// 自定义插槽容器
.tree-node-wrapper {
  line-height: 22px;
  vertical-align: middle;
}

// 文本内容
.tree-node-title,
.product-title {
  line-height: 22px;
  vertical-align: baseline;
}
```

#### 3. 图标垂直对齐
```less
// 切换器图标（展开/收起）
.ant-tree-switcher {
  display: inline-flex;
  align-items: center;
  vertical-align: middle;
  
  .ant-tree-switcher-line-icon {
    vertical-align: middle;
  }
}

// 节点图标
.ant-tree-iconEle {
  display: inline-flex;
  align-items: center;
  vertical-align: middle;
  
  svg {
    vertical-align: middle;
  }
}
```

#### 4. 自定义内容使用 inline-block
```less
.product-tree-node {
  display: inline-block;
  line-height: 22px;
  vertical-align: middle;
}
```

### 注意事项
- **关键点**：`.ant-tree-treenode` 必须使用 `display: flex` + `align-items: center`
- 所有文本和容器统一使用相同的 `line-height`（建议 22px）
- 图标元素使用 `inline-flex` + `vertical-align: middle`
- 避免使用多层 flex 嵌套，可能导致对齐问题
- 自定义插槽内容使用 `inline-block` 而不是 `flex`
