# 上传进度优化说明

## 优化概述
为上传文件组件添加了**实时百分比显示**、**已持续时间显示**、**剩余时间预估**和**错误容错机制**功能，提升用户体验和系统稳定性。

### 2. 新增功能

#### 2.1 进度百分比显示
- **位置**: 第 61-70 行
- **显示内容**:
  - 当前进度百分比（精确到小数点后 2 位）
  - 已处理数量 / 总数量

#### 2.2 已持续时间显示
- **位置**: 第 64-67 行
- **显示内容**: 从开始上传到当前已经持续的时间
- **更新机制**: 每秒自动刷新
- **实现方式**:
  - 使用独立的定时器 `elapsedTimeTimer` 每秒更新
  - 通过响应式变量 `currentTime` 触发视图更新
  - 颜色标识：橙色（#fa8c16）

#### 2.3 剩余时间预估
- **计算逻辑**: 第 283-289 行
- **算法说明**:
  ```typescript
  已用时间 = 当前时间 - 开始时间
  处理速率 = 已处理数量 / 已用时间
  剩余数量 = 总数量 - 已处理数量
  预估剩余时间 = 剩余数量 / 处理速率
  ```
- **显示格式**:
  - 小于 60 秒: "X秒"
  - 60-3600 秒: "X分Y秒"
  - 超过 3600 秒: "X小时Y分钟"

#### 2.4 错误容错机制
- **位置**: 第 235-237 行、第 404-416 行
- **功能说明**: 防止定时任务因网络波动或服务异常导致的无限重试
- **机制**:
  - 记录进度查询连续失败次数
  - 达到 5 次连续失败后自动停止所有定时器
  - 向用户显示友好的错误提示
  - 成功获取进度后自动重置错误计数

- 文件上传

所有需要异步处理并展示进度的场景都会自动显示百分比和剩余时间。

## 技术要点
1. **时间戳精度**: 使用 `Date.now()` 获取毫秒级时间戳
2. **动态计算**: 每次进度更新时重新计算剩余时间，确保预估准确性
3. **实时更新**: 已持续时间每秒自动刷新，无需等待进度更新
4. **响应式设计**: 通过 `ref(currentTime)` 实现视图的响应式更新
5. **条件显示**: 
   - 已持续时间：当 `config.startTime > 0` 时显示
   - 剩余时间：当 `config.estimatedTime > 0` 时显示
6. **状态管理**: 在不同步骤切换时正确维护时间相关状态和定时器
7. **定时器管理**: 
   - `uploadProgressTimer`: 每 2 秒更新一次进度数据
   - `elapsedTimeTimer`: 每 1 秒更新一次已持续时间
8. **错误容错机制**: 
   - 使用 Promise 的 `.catch()` 捕获异步错误
   - 连续失败计数器防止无限重试
   - 达到阈值后自动停止，避免资源浪费
   - 成功后自动重置，保证正常流程不受影响

## 注意事项
1. 剩余时间为预估值，实际时间可能因网络波动、服务器处理速度等因素有所差异
2. 时间计算基于线性速率假设，适用于均匀处理的场景
3. 如果处理速率不稳定，预估时间可能会有较大波动
4. 已持续时间每秒刷新一次，轻量级定时器不会影响性能
5. 所有定时器在组件状态变更时都会正确清理，避免内存泄漏
6. **错误容错**:
   - 进度查询连续失败 5 次后会自动停止轮询
   - 用户会收到明确的错误提示
   - 建议用户刷新页面重试或联系管理员
   - 错误日志会记录在控制台，便于问题排查
