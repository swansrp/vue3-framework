# GroupManage 组件动态插槽与绑定信息扩展说明

## 功能概述

本次修改为 `GroupManage` 组件添加了动态插槽支持和绑定信息扩展功能，允许使用者自定义每个 tab 的渲染方式，并支持在绑定关系之上附加额外的业务信息（如权限控制）。

## 核心改动

### 1. API 扩展

#### 新增接口方法

在 `src/framework/apis/index.ts` 和 `src/framework/apis/portal/index.ts` 中新增：

- **getBindInfoRequest**: 获取绑定信息的底层请求方法
- **getBindInfo**: 对外暴露的获取绑定信息方法

```typescript
// 获取绑定信息
export const getBindInfo = (
  entity: string, 
  attach: string, 
  entityId: any, 
  attachId: any, 
  baseDomain?: string, 
  showSuccess = false, 
  showLoading = false
) => getBindInfoRequest('/' + entity + '/' + attach, entityId, attachId, baseDomain, showSuccess, showLoading)
```

### 2. 组件类型扩展

在 `GroupBindProperty` 接口中新增字段：

```typescript
export interface GroupBindProperty {
    // ... 原有字段
    
    // 是否支持绑定信息编辑
    supportBindInfo?: boolean
    // 绑定信息的映射 key为attachId value为bindInfo对象
    bindInfoMap?: Map<any, any>
}
```

### 3. 动态插槽机制

#### 插槽命名规范

组件支持以下插槽（`tabKey` 为每个 tab 的唯一标识）：

- **`tab_${tabKey}`**: 完全接管整个 tab 内容（优先级最高，提供后 `readOnly`、`treeMode` 等字段将被忽略）
- **`tabKey`**: readonly 模式下的列表项/树节点渲染
- **`view_${tabKey}`**: 查看模式下的已绑定项渲染
- **`bind_${tabKey}`**: 绑定模式下的可选项渲染

> GroupManage 内部通过 `useSlots()` 自动检测父组件是否提供了 `tab_${tabKey}` 插槽，有则走具名插槽模式，无则走默认 list / tree 逻辑。

#### 插槽传递参数

每个插槽都会接收以下参数：

| 参数名 | 类型 | 说明 |
|--------|------|------|
| data | Array | 数据列表（待绑定项或已绑定项） |
| checked | Array | 当前选中的值列表（仅 bind 插槽） |
| onCheck | Function | 选中变更回调（仅 bind 插槽） |

### 4. 绑定信息自动加载

当 `GroupBindProperty.supportBindInfo = true` 时，组件会在加载已绑定列表后：

1. 自动调用 `getBindInfo` 获取每个绑定项的详细信息
2. 将绑定信息存储到 `bindInfoMap` 中
3. 将绑定信息合并到显示数据中

```typescript
if (bindTab.supportBindInfo && bindTab.bindData.length > 0) {
  bindTab.bindInfoMap = new Map()
  const [entity, attach] = bindTab.baseUrl.split('/')
  
  const promises = bindTab.bindData.map((item: any) => {
    const attachId = item[bindTab.bindDataValueField]
    return getBindInfo(entity, attach, currentUserGroupInfo.value.id, attachId)
      .then((bindInfoResp: any) => {
        if (bindInfoResp && bindInfoResp.payload) {
          bindTab.bindInfoMap.set(attachId, bindInfoResp.payload)
          Object.assign(item, bindInfoResp.payload)
        }
      })
  })
  
  await Promise.all(promises)
}
```

### 5. 数据共享机制

使用 Vue 3 的 `provide/inject` 机制共享数据：

```typescript
// GroupManage 组件中
provide('currentUserGroupId', currentUserGroupInfo)
provide('bindTabs', bindTabs)

// 使用方中
const currentUserGroupInfo = inject<Ref<IdName>>('currentUserGroupId')
const bindTabs = inject<Array<GroupBindProperty>>('bindTabs')
```

## 使用示例

### 基础配置

```typescript
const bindTabList: Array<GroupBindProperty> = [{
  tabKey: 'dept',
  title: '归属部门',
  baseUrl: 'group/hr/dept',
  dict: 'SECONDARY_DEPT_DICT',
  bindDataValueField: 'value',
  bindDataDisplayField: 'label',
  supportBindInfo: true  // 启用绑定信息功能
}]
```

### 自定义渲染

```vue
<template>
  <group-manage :bind-tab-list="bindTabList">
    <!-- 自定义查看模式 -->
    <template #view_dept="{ data }">
      <a-list :data-source="data" bordered size="small">
        <template #renderItem="{ item }">
          <a-list-item>
            <div style="display: flex; justify-content: space-between; width: 100%;">
              <span>{{ item.label }}</span>
              <a-tag :color="item.readOnly ? 'red' : 'green'">
                {{ item.readOnly ? '只读' : '可编辑' }}
              </a-tag>
            </div>
          </a-list-item>
        </template>
      </a-list>
    </template>

    <!-- 自定义绑定模式 -->
    <template #bind_dept="{ data, checked, onCheck }">
      <a-checkbox-group :value="checked" @change="onCheck">
        <div v-for="item in data" :key="item.value">
          <a-checkbox :value="item.value">{{ item.label }}</a-checkbox>
          <!-- 仅已选中项显示权限开关 -->
          <a-switch
            v-if="checked.includes(item.value)"
            v-model:checked="bindInfoMap.get(item.value).readOnly"
            @change="handleReadOnlyChange(item.value, $event)"
          />
        </div>
      </a-checkbox-group>
    </template>
  </group-manage>
</template>
```

### 更新绑定信息

```typescript
const handleReadOnlyChange = (attachId: any, readOnly: boolean) => {
  const [entity, attach] = bindTabList[0].baseUrl.split('/')
  updateBindInfo(entity, attach, userGroupId, attachId, { readOnly }, false)
    .then(() => message.success('权限更新成功'))
}
```

## 后端接口要求

### 查看绑定信息

```
GET /{entity}/{attach}/bind/info?entityId={}&attachId={}
```

返回格式：
```json
{
  "payload": {
    "readOnly": false
  }
}
```

### 修改绑定信息

```
POST /{entity}/{attach}/bind/info?entityId={}&strict=false
```

请求体：
```json
{
  "attachId": "xxx",
  "data": {
    "readOnly": true
  }
}
```

## 技术要点

1. **插槽作用域**：通过具名插槽的 scope 参数传递数据和回调
2. **响应式更新**：bindInfoMap 存储在响应式对象中，支持双向绑定
3. **异步加载**：使用 Promise.all 并发加载所有绑定信息
4. **默认渲染**：插槽内提供默认实现，未定义插槽时自动降级
5. **数据注入**：通过 provide/inject 跨组件共享状态

## 适用场景

- 需要在绑定关系上附加额外属性（如权限、备注、配置）
- 需要自定义绑定项的展示样式
- 需要在绑定界面添加额外操作（如批量设置权限）
- 不同 tab 需要完全不同的渲染逻辑
- 需要完全独立的自定义 tab 内容（非 list / tree），通过 `tab_${tabKey}` 插槽实现，如表格、图表、表单等复杂交互界面

## 注意事项

1. `supportBindInfo` 会增加接口调用次数，仅在需要时启用
2. 绑定信息更新后需手动刷新列表或通过 key 强制重新渲染
3. 插槽中访问 bindInfoMap 需确保数据已加载完成
4. 使用 inject 时需检查数据是否存在，避免运行时错误
5. `tab_${tabKey}` 插槽优先级最高，提供后 `readOnly`、`treeMode` 等字段将被忽略，数据加载和渲染完全由父组件控制
