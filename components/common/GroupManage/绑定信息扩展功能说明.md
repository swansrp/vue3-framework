# GroupManage 组件绑定信息扩展功能说明

## 功能概述

本次优化为 `GroupManage` 组件增加了对绑定数据的额外属性管理功能，允许在查看tab中对已绑定的数据进行属性扩展（如 `readOnly` 等），实现更灵活的权限和配置管理。

## 主要变更

### 1. 类型定义扩展 (`types.ts`)

在 `GroupBindProperty` 接口中新增两个属性：

```typescript
export interface GroupBindProperty {
    // ... 其他属性
    
    // 支持绑定信息扩展（如 readOnly 等额外属性）
    supportBindInfo?: boolean
    
    // 绑定信息 Map，key 为 attachId，value 为绑定信息对象
    bindInfoMap?: Map<any, any>
}
```

### 2. API 调用优化

将原有的 `getBindInfo(entity, attach, ...)` 方式统一改为使用新的 URL 方式：

- `getBindInfoByUrl(baseUrl, entityId, attachId)` - 获取绑定信息
- `updateBindInfoByUrl(baseUrl, entityId, attachId, data)` - 更新绑定信息

### 3. 组件功能增强

#### 3.1 数据加载时自动获取绑定信息

当 `supportBindInfo: true` 时，组件会在加载绑定数据后自动获取每个绑定项的额外信息：

```javascript
if (bindTab.supportBindInfo && bindTab.bindData.length > 0) {
  bindTab.bindInfoMap = new Map()
  
  const promises = bindTab.bindData.map((item: any) => {
    const attachId = item[bindTab.bindDataValueField]
    return getBindInfoByUrl(bindTab.baseUrl, currentUserGroupInfo.value.id, attachId)
      .then((bindInfoResp: any) => {
        if (bindInfoResp && bindInfoResp.payload) {
          bindTab.bindInfoMap.set(attachId, bindInfoResp.payload)
          Object.assign(item, bindInfoResp.payload)
        }
      })
  })
  
  Promise.all(promises).then(() => {
    // 刷新视图
  })
}
```

#### 3.2 提供更新绑定信息的方法

通过 `provide/inject` 机制向父组件提供 `updateBindInfo` 方法：

```javascript
const updateBindInfoData = (tabKey: string, attachId: any, data: any) => {
  const bindTab = bindTabs.find(tab => tab.tabKey === tabKey)
  if (!bindTab || !bindTab.supportBindInfo) {
    return Promise.reject('Tab not found or does not support bind info')
  }
  
  return updateBindInfoByUrl(bindTab.baseUrl, currentUserGroupInfo.value.id, attachId, data)
    .then((resp: any) => {
      // 更新本地缓存
      if (bindTab.bindInfoMap) {
        const currentInfo = bindTab.bindInfoMap.get(attachId) || {}
        bindTab.bindInfoMap.set(attachId, { ...currentInfo, ...data })
      }
      
      // 更新 bindData 中的数据
      const item = bindTab.bindData?.find((i: any) => i[bindTab.bindDataValueField] === attachId)
      if (item) {
        Object.assign(item, data)
      }
      
      // 刷新视图
      bindTab.key = !bindTab.key
      
      return resp
    })
}

provide('updateBindInfo', updateBindInfoData)
```

## 使用示例

### 父组件配置

```vue
<template>
  <group-manage :bind-tab-list="bindTabList">
    <template #view_dept="{ data }">
      <a-list
        v-if="data && data.length > 0"
        :data-source="data"
        bordered
        size="small"
      >
        <template #renderItem="{ item }">
          <a-list-item>
            <div style="display: flex; justify-content: space-between; align-items: center; width: 100%;">
              <span>{{ item.label }}</span>
              <div style="display: flex; align-items: center; gap: 10px;">
                <a-tag :color="item.readOnly ? 'red' : 'green'">
                  {{ item.readOnly ? '只读' : '可编辑' }}
                </a-tag>
                <a-button 
                  type="link" 
                  size="small"
                  @click="toggleReadOnly(item)"
                >
                  {{ item.readOnly ? '设置为可编辑' : '设置为只读' }}
                </a-button>
              </div>
            </div>
          </a-list-item>
        </template>
      </a-list>
      <a-empty v-else />
    </template>
  </group-manage>
</template>

<script lang="ts" setup>
import { inject } from 'vue'
import { message } from 'ant-design-vue'
import GroupManage from '@/framework/components/common/GroupManage/index.vue'
import { GroupBindProperty } from '@/framework/components/common/GroupManage/types'

const bindTabList: Array<GroupBindProperty> = [{
  tabKey: 'dept',
  title: '归属部门',
  baseUrl: 'group/hr/dept',
  dict: 'SECONDARY_DEPT_DICT',
  bindDataValueField: 'value',
  bindDataDisplayField: 'label',
  supportBindInfo: true, // 启用绑定信息支持
}]

// 注入 GroupManage 组件提供的更新方法
const updateBindInfo = inject<(tabKey: string, attachId: any, data: any) => Promise<any>>('updateBindInfo')

// 切换只读状态
const toggleReadOnly = (item: any) => {
  if (!updateBindInfo) {
    message.error('更新功能未就绪')
    return
  }
  
  const newReadOnly = !item.readOnly
  
  updateBindInfo('dept', item.value, { readOnly: newReadOnly })
    .then(() => {
      message.success(`已设置为${newReadOnly ? '只读' : '可编辑'}`)
    })
    .catch((error: any) => {
      message.error('更新失败：' + (error.message || error))
    })
}
</script>
```

## 技术要点

### 1. 插槽命名规范

查看tab使用动态插槽命名：`view_${tabKey}`，遵循现有的命名规范。

### 2. 数据流

```
用户选择用户组
  ↓
加载绑定数据 (getAllBindListByUrl)
  ↓
如果 supportBindInfo = true
  ↓
并行获取每个绑定项的信息 (getBindInfoByUrl)
  ↓
合并信息到 bindData 中
  ↓
父组件通过插槽渲染 + 提供交互
  ↓
用户操作触发更新 (updateBindInfo)
  ↓
本地数据同步更新 + 视图刷新
```

### 3. 性能优化

- 使用 `Promise.all` 并行获取绑定信息，避免串行等待
- 本地缓存绑定信息到 `bindInfoMap`，减少重复请求
- 更新时同步更新本地缓存和视图数据

### 4. 错误处理

- API 请求失败时设置默认值，确保界面正常显示
- 提供友好的错误提示信息

## 具名插槽模式（完全自定义 tab 内容）

除了 list 和 tree 两种渲染方式外，父组件还可以通过提供 `tab_${tabKey}` 具名插槽，完全接管某个 tab 的内容渲染。

### 插槽命名规则

GroupManage 内部通过 `useSlots()` 自动检测父组件是否提供了 `tab_${tabKey}` 插槽：

- 有 → 走具名插槽模式，整个 tab 内容由插槽渲染
- 无 → 走默认的 list / tree / readOnly 逻辑

> 注意：`tab_${tabKey}` 插槽用于**整个 tab 内容**的替换，与 readOnly/default 模式下的 `view_${tabKey}`、`bind_${tabKey}` 等**列表项级**插槽不冲突。

### 插槽传递参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| bindTab | GroupBindProperty | 当前 tab 的完整配置 |
| userGroupInfo | IdName | 当前选中的用户组信息 |

### 完整示例

#### 第一步：编写自定义内容组件

```vue
<!-- src/views/bid/components/apiKeyAssign/index.vue -->
<template>
  <div>
    <h3>用户组：{{ userGroupInfo?.name }}</h3>
    <!-- 你自己的组件内容 -->
  </div>
</template>

<script lang="ts" setup>
import { GroupBindProperty } from '@/framework/components/common/GroupManage/types'
import { IdName } from '@/framework/utils/type'

// GroupManage 具名插槽模式注入的 props
const props = defineProps<{
  bindTab: GroupBindProperty
  userGroupInfo: IdName
}>()
</script>
```

#### 第二步：在父组件中使用 `tab_${tabKey}` 插槽

```vue
<!-- src/views/bid/userGroup/index.vue -->
<template>
  <group-manage :bind-tab-list="bindTabList">
    <!-- 插槽名 = "tab_" + tabKey -->
    <template #tab_api_key="slotProps">
      <ApiKeyAssign
        :bind-tab="(slotProps as any).bindTab"
        :user-group-info="(slotProps as any).userGroupInfo"
      />
    </template>

    <!-- 原有的 dept 查看插槽（列表项级别，不受影响） -->
    <template #view_dept="{ item, updateBindInfo }">
      <div style="display: flex; justify-content: space-between; width: 100%;">
        <span>{{ item.label }}</span>
        <a-tag :color="item.readOnly === '1' ? 'red' : 'green'">
          {{ item.readOnly === '1' ? '只读' : '可编辑' }}
        </a-tag>
      </div>
    </template>
  </group-manage>
</template>

<script lang="ts" setup>
import GroupManage from '@/framework/components/common/GroupManage/index.vue'
import { GroupBindProperty } from '@/framework/components/common/GroupManage/types'
import ApiKeyAssign from '@/views/bid/components/apiKeyAssign/index.vue'

const bindTabList: Array<GroupBindProperty> = [
  // 具名插槽模式：只需声明 tabKey，GroupManage 自动检测父组件的 #tab_api_key 插槽
  {
    tabKey: 'api_key',
    title: 'API Key分配',
    baseUrl: '',              // 不需要可传空
    bindDataValueField: '',   // 不需要可传空
    bindDataDisplayField: '', // 不需要可传空
  },
  // 普通 list 模式 + 绑定信息扩展
  {
    tabKey: 'dept',
    title: '归属部门',
    baseUrl: 'group/hr/dept',
    dict: 'SECONDARY_DEPT_DICT',
    bindDataValueField: 'value',
    bindDataDisplayField: 'label',
    supportBindInfo: true,
    defaultBindInfo: { readOnly: '0' },
  },
]
</script>
```

### 关键点说明

| 配置项 | 说明 |
|--------|------|
| 插槽名 `tab_${tabKey}` | 父组件提供此插槽即自动生效，无需额外配置字段 |
| `slotProps.bindTab` | 当前 tab 的完整配置对象 |
| `slotProps.userGroupInfo` | 当前选中用户组信息，用于响应切换 |
| `baseUrl` / `bindDataValueField` / `bindDataDisplayField` | 具名插槽模式下传空字符串即可 |

### 注意事项

- 具名插槽模式优先级最高，提供 `tab_${tabKey}` 插槽后 `readOnly`、`treeMode` 等字段将被忽略
- 该模式下，选择用户组时不会自动加载绑定数据，由插槽内容自行管理
- 不会显示全选/清空等批量操作按钮
- `onMounted` 中不会自动加载字典数据

## 扩展性

该架构支持扩展任意属性，不仅限于 `readOnly`：

```typescript
// 可以扩展更多属性
updateBindInfo('dept', item.value, {
  readOnly: true,
  priority: 1,
  remark: '备注信息',
  customField: 'value'
})
```

## 注意事项

1. **启用条件**：需要在 `bindTabList` 中设置 `supportBindInfo: true`
2. **API 支持**：后端需要实现对应的 `getBindInfo` 和 `updateBindInfo` 接口
3. **数据安全**：获取绑定信息失败时会使用默认值，确保界面不会崩溃
4. **视图刷新**：更新后通过切换 `key` 属性强制刷新视图
5. **具名插槽模式**：提供 `tab_${tabKey}` 插槽后，数据加载和渲染完全由父组件控制，框架不干预
