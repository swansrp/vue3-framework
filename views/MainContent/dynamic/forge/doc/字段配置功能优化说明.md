# 字段配置功能优化说明

## 修改日期
2025-11-26

## 问题描述
列配置中缺少"是否显示"字段的渲染,导致无法在界面上直接控制字段的显示状态。

## 修改内容

### 1. 类型定义扩展
**文件**: `src/views/forge/types.ts`

在 `ColumnInfo` 接口中新增字段:
``typescript
isVisible?: string // 是否显示
```

### 2. 表格列配置
**文件**: `src/views/forge/components/ColumnConfig.vue`

#### 2.1 新增表格列
在 `columns` 数组中添加:
```typescript
{ title: '是否显示', dataIndex: 'isVisible', key: 'isVisible', width: 100, align: 'center' }
```

#### 2.2 新增表格单元格渲染
使用开关组件渲染"是否显示"字段,支持直接切换:
```vue
<template v-else-if="column.key === 'isVisible'">
  <a-switch
    :checked="record.isVisible === '1'"
    @change="(checked) => handleVisibleChange(record, checked)"
  />
</template>
```

#### 2.3 新增切换处理函数
``typescript
const handleVisibleChange = async (record: ColumnInfo, checked: boolean) => {
  try {
    await sysMatrixColumnUpdate({}, {
      ...record,
      isVisible: checked ? '1' : '0'
    } as any)
    
    record.isVisible = checked ? '1' : '0'
    message.success('更新成功')
  } catch (error) {
    console.error('更新失败:', error)
    await loadData()
  }
}
```

### 3. 表单配置扩展
在字段编辑表单中新增"是否显示"复选框:

``vue
<a-row :gutter="16">
  <a-col :span="12">
    <a-form-item
      label="是否显示"
      name="isVisible"
      :label-col="{ span: 10 }"
      :wrapper-col="{ span: 14 }"
    >
      <a-checkbox
        :checked="formData.isVisible === '1'"
        @change="(e: any) => formData.isVisible = e.target.checked ? '1' : '0'"
      />
    </a-form-item>
  </a-col>
</a-row>
```

### 4. 默认值设置
在新增字段时,默认将 `isVisible` 设置为 `'1'` (显示):
``typescript
isVisible: '1'
```

## 功能特性

1. **直接切换**: 在表格中通过开关组件直接切换字段的显示状态
2. **即时保存**: 切换后立即调用 API 保存到后端
3. **UI反馈**: 操作成功后显示提示消息
4. **错误恢复**: 如果更新失败,自动重新加载数据确保数据一致性
5. **默认显示**: 新增字段时默认为显示状态

## UI展示方式

- **表格列**: 使用 `a-switch` 开关组件,支持点击切换
- **编辑表单**: 使用 `a-checkbox` 复选框,符合用户偏好的布尔字段展示方式

## 注意事项

1. 字段值使用 `'1'` 表示显示,`'0'` 表示不显示
2. 切换操作会立即触发 API 调用,确保数据实时保存
3. 与 dataset 模块的列配置保持功能一致性

## 二、表单配置组件优化 (FormConfig.vue)

### 1. 关联字段选择过滤

**需求**：主键字段不应出现在关联字段下拉列表

**实现**：
``vue
<a-select-option
  v-for="col in columnList.filter(c => c.isPrimaryKey !== '1')"
  :key="col.id"
  :value="col.id"
>
  {{ col.columnComment }}（{{ col.columnName }}）
</a-select-option>
```

**效果**：
- 过滤掉主键字段
- 避免误配置

---

### 2. 自动填充显示标签

**需求**：选择关联字段后，自动填充标签名

**实现**：
```javascript
const handleColumnChange = (columnId) => {
  const selectedColumn = columnList.value.find(col => col.id === columnId)
  if (selectedColumn && !formData.id) {
    // 只在新增时自动填充
    formData.label = selectedColumn.columnComment || selectedColumn.columnName
  }
}
```

**效果**：
- 提升配置效率
- 减少重复输入
- 编辑时不覆盖已有值

## 五、使用指南

### 添加普通字段流程

1. 点击"添加字段"按钮
2. 输入字段注释（中文）→ 失去焦点后自动生成字段名
3. 选择数据库类型和表单类型
4. 设置默认值或引用序列（二选一）
5. 根据需要勾选：是否可空、是否索引、是否唯一
6. 如需特殊标记，勾选：名称字段/顺序字段/父节点字段（三选一）

### 添加主键字段流程

1. 点击"添加字段"按钮
2. 字段名输入 `id`，字段注释输入"主键ID"
3. 数据库类型选择 `BIGINT`
4. 勾选"是否主键"
5. 在引用序列框中点击"设为自增"按钮 → 填充 `AUTO_INCREMENT`

### 设置NULL默认值流程

1. 勾选"是否可空" ✓
2. 点击"设为NULL"按钮（自动联动勾选"是否可空"）
3. 默认值框显示 `null`

## 六、注意事项

### 1. 字段命名规范
- ✅ 使用小写字母、数字、下划线
- ✅ 以字母开头
- ❌ 不要使用大写字母
- ❌ 不要以数字或下划线开头

### 2. 互斥字段配置
- 默认值 vs 引用序列（二选一）
- 关联矩阵 vs 关联字典（二选一）
- 名称字段 vs 顺序字段 vs 父节点字段（三选一）

### 3. 主键字段特殊性
- 不显示默认值字段
- 显示引用序列字段
- 通常设置为自增

### 4. NULL默认值限制
- 必须先勾选"是否可空"
- 未勾选时按钮禁用
