# 动态矩阵配置系统

## 功能概述

动态矩阵配置系统允许通过可视化界面配置数据库表、字段、表单和联动逻辑，无需编写代码即可实现完整的数据管理功能。

## 使用流程

### 1. 创建矩阵

1. 进入“矩阵管理”标签页
2. 点击“新建矩阵”按钮
3. 填写矩阵信息：
   - **表名**：数据库表名（必须以 `biz_` 开头，如：`biz_user_info`）
   - **表注释**：表的中文说明
   - **数据源**：默认 `master`
   - **主键字段**：默认 `id`
   - **存储引擎**：推荐 `InnoDB`
   - **字符集**：推荐 `utf8mb4`
4. 点击确定保存

> ✨ **新特性**：表名前缀 `biz_` 为不可编辑部分，用户只需输入后缀，防止误删

### 2. 配置字段

1. 在矩阵列表中点击"字段配置"按钮
2. 点击"添加字段"
3. 配置字段信息：
   - **字段名**：英文字段名（如：`user_name`）
     - 必须以小写字母开头
     - 只能包含小写字母、数字和下划线
   - **字段注释**：中文说明（如：用户姓名）
     - 失去焦点时自动转拼音生成字段名（仅新增时）
   - **数据库类型**：选择合适的类型
     - `VARCHAR` - 短文本（需设置长度，默认50）
     - `TEXT` - 长文本
     - `INT` - 整数
     - `BIGINT` - 长整数
     - `DECIMAL` - 小数（需设置总位数和小数位）
     - `DATE` - 日期
     - `DATETIME` - 日期时间
   - **表单类型**：选择前端展示类型（自动映射到表单配置）
   - **默认值/引用序列**：
     - 非主键字段：二选一（填写一个会清空另一个）
     - 主键字段：只显示引用序列，提供“设为自增”按钮
   - **是否可空**：字段是否允许为空
     - 点击“设为NULL”按钮会自动勾选此项
     - 未勾选时“设为NULL”按钮禁用
   - **是否主键**：是否为主键字段
   - **是否索引**：是否创建索引（常用查询字段建议创建）
   - **是否唯一**：是否创建唯一索引
   - **是否显示**：是否在表单中显示
   - **名称字段/顺序字段/父节点字段**：三选一互斥模式
     - 勾选一个会自动取消其他两个
     - 允许全部为“否”
   - **关联矩阵/关联字典**：用于下拉选择等类型
     - 两者填写互斥，填写一个会清空另一个
4. 使用"上移"/"下移"调整字段顺序
5. 配置完所有字段后，关闭抽屉

> ✨ **新特性**：关闭字段配置抽屉时，矩阵列表会自动刷新，确保按钮状态实时更新
> ✨ **中文转拼音**：输入中文字段注释后失去焦点，系统自动将中文转为拼音生成字段名（仅新增时）
> ✨ **主键字段逻辑**：主键字段不显示默认值，只显示引用序列，提供“设为自增”快捷按钮
> ✨ **互斥字段**：
> - 默认值 ↔ 引用序列（非主键）
> - 名称字段 ↔ 顺序字段 ↔ 父节点字段
> - 关联矩阵 ↔ 关联字典

### 3. 创建物理表

1. 在矩阵列表中找到配置好的矩阵
2. 点击状态列的创建图标或“创建表”按钮（需要先配置字段）
3. 如果没有配置主键字段，系统会弹出确认框
4. 系统会根据字段配置在数据库中创建真实的表
5. 创建成功后，状态变为“已创建”（蓝色）

> ✨ **新特性**：
> - 没有字段时创建图标自动置灰禁用，鼠标悬停显示提示“请先配置字段”
> - 状态可视化：
>   - 未创建（灰色）- 显示创建图标
>   - 已创建（蓝色）- 隐藏操作按钮
>   - 已同步（绿色）- 隐藏操作按钮
>   - 待同步（橙色）- 显示红色同步图标
> - 创建表和同步表结构为互斥显示，不会同时出现
> - 没有主键时会弹出确认框，提示用户是否继续

### 4. 配置表单

1. 点击“表单配置”按钮（需要先配置字段）
2. 在“表单配置”标签页中点击“添加表单字段”
3. 配置表单展示信息：
   - **关联字段**：选择对应的数据库字段
   - **显示标签**：表单中显示的标签文字
   - **描述信息**：提示用户如何填写
   - **字段类型**：前端组件类型
     - 单行文本、多行文本
     - 数字、真值（开关）
     - 下拉选择、树形下拉
     - 日期、日期时间
     - 图片、文件
     - 货币、百分比
   - **是否必填**：是否为必填项
   - **是否只读**：是否只读
   - **验证规则**：正则表达式验证
   - **最小值/最大值**：数值范围限制
   - **字典编码**：下拉选择等类型的字典
   - **单位**：显示的单位（如：元、%）
4. 切换到“布局配置”标签页，可以拖拽调整字段位置和大小
5. 为所有需要展示的字段创建表单配置

> ✨ **新特性**：
> - 没有字段时“表单配置”按钮自动禁用
> - **可视化布局设计**：支持拖拽调整字段位置和尺寸
> - **双标签页系统**：表单配置 + 布局配置 + 联动配置

### 5. 配置联动（可选）

1. 在"表单配置"页面切换到"表单联动配置"标签页
2. 点击"添加联动规则"
3. 配置联动逻辑：
   - **触发字段**：哪个字段变化时触发
   - **联动名称**：规则说明
   - **触发事件**：值改变/失去焦点/获得焦点
   - **条件脚本**：什么条件下执行（可选）
   - **动作脚本**：执行的JavaScript代码
   - **目标字段**：哪些字段会被更新
   - **优先级**：多个规则的执行顺序

#### 联动脚本示例

**计算总分：**
```javascript
// 条件脚本
score1 != null && score2 != null && score3 != null

// 动作脚本
totalScore = parseFloat(score1) + parseFloat(score2) + parseFloat(score3);
avgScore = totalScore / 3;
```

**根据年龄判断分组：**
```javascript
// 条件脚本
age != null

// 动作脚本
if (age < 18) {
  ageGroup = '未成年';
} else if (age < 60) {
  ageGroup = '成年人';
} else {
  ageGroup = '老年人';
}
```

**价格折扣计算：**
```javascript
// 条件脚本
originalPrice != null && discountRate != null

// 动作脚本
finalPrice = originalPrice * (1 - discountRate / 100);
discountAmount = originalPrice - finalPrice;
```

4. 点击"测试"按钮可以测试联动逻辑是否正确

### 6. 同步表结构

当字段配置发生变更后（添加、删除或修改字段）：

1. 表状态会变为“待同步”（橙色）
2. “同步表结构”按钮变为红色主按钮，显示“立即同步”
3. 点击后系统会执行 DDL 语句更新数据库表结构
4. 同步成功后状态变为“已同步”（绿色）

> ✨ **新特性**：
> - 只有待同步状态时才能点击同步按钮
> - 已创建和已同步状态下同步按钮禁用，防止不必要的操作

### 7. 查看变更历史

1. 在矩阵列表中点击“变更历史”按钮（仅已创建的表可用）
2. 以时间线形式展示表结构的所有变更记录
3. 每个变更显示：
   - **版本号**：变更的版本
   - **变更类型**：创建表/添加字段/调整字段顺序/添加索引/删除索引/删除字段
   - **影响字段**：变更涉及的字段名
   - **执行状态**：成功/失败
   - **DDL 语句**：可展开查看具体执行的 SQL
   - **错误信息**：如果失败会显示错误详情
4. 按版本分组显示，最新版本在最上方

> ✨ **新功能**：变更历史追溯
> - 完整记录表结构的演变过程
> - 支持查看 DDL 语句，方便问题排查
> - 失败记录红色高亮显示，并显示错误信息

### 8. 清空表数据

1. 在矩阵列表中点击“清空数据”按钮（仅已创建的表可用）
2. 二次确认弹窗
3. 点击“确定”清空表中所有数据（不删除表结构）

> ⚠️ **危险操作**：清空后数据无法恢复，请慎重操作

### 9. 数据管理

1. 在矩阵列表中点击"数据管理"按钮，或者切换到"数据管理"标签页
2. 点击"新增数据"填写表单
3. 表单会自动根据配置展示相应的字段和组件
4. 填写数据时，配置的联动规则会自动执行
5. 支持编辑、查看、删除操作

## 注意事项

### 表名和字段名规范

- **表名前缀**：必须以 `biz_` 开头（系统自动添加，不可删除）
- **命名规范**：使用小写字母和下划线
- **避免关键字**：不要使用MySQL关键字
- **建议格式**：`模块_功能_说明`（如：`biz_user_info`）

### 字段类型选择建议

- **主键**：使用 `BIGINT` + 自增
- **金额**：使用 `DECIMAL(19,2)`
- **百分比**：使用 `DECIMAL(5,2)`
- **短文本**：使用 `VARCHAR(适当长度)`
- **长文本**：使用 `TEXT`
- **时间戳**：使用 `DATETIME`

### 索引创建建议

- 为常用查询字段添加索引
- 避免在大文本字段上创建索引
- 唯一性约束的字段使用唯一索引

### 联动脚本注意事项

- 使用标准JavaScript语法
- 可以访问表单中所有字段（通过字段名）
- 避免使用浏览器API（window、document等）
- 脚本在服务端执行，确保安全性

## 常见问题

### Q: 创建表失败怎么办？

A: 检查以下几点：
1. 是否已配置至少一个字段
2. 字段名是否符合规范
3. 是否有重复的表名
4. 数据库连接是否正常

### Q: 如何修改已创建的表？

A: 
1. 如果表已创建，可以继续添加新字段
2. 添加新字段后，点击"同步表结构"按钮
3. 已存在的字段无法修改，只能删除重建

### Q: 联动不生效怎么办？

A:
1. 检查联动规则是否启用
2. 使用"测试"功能验证脚本逻辑
3. 检查条件脚本是否正确
4. 检查字段名是否匹配

### Q: 如何导入导出数据？

A: 后续版本将支持Excel导入导出功能
