# 动态矩阵配置系统 - 功能特性说明

## 核心特性

### 1. 表名前缀保护

**biz_前缀不可编辑**:
- 用户输入:`user_info` → 系统保存:`biz_user_info`
- 编辑时只能修改后缀部分,前缀`biz_`固定不可删除
- 防止用户误删前缀导致表名不符合规范

**好处**:
- 统一业务表命名规范
- 与系统表区分开来(sys_、t_等)
- 防止误操作

### 2. 智能字段配置

**中文转拼音**:
- 输入字段注释(中文)后失去焦点
- 系统自动将中文转换为拼音生成字段名
- 示例:"用户姓名" → "yong_hu_xing_ming" 或 "user_name"
- 仅在新增字段时生效,编辑时不会覆盖

**字段名规范**:
- 必须以小写字母开头
- 只能包含小写字母、数字和下划线
- 前端正则验证:`/^[a-z][a-z0-9_]*$/`

**互斥字段规则**:

*三选一互斥(名称/顺序/父节点)*:
- 最多只能选中一个
- 勾选一个会自动取消其他两个
- 允许全部为"否"

*二选一互斥*:
- **默认值 ⟷ 引用序列**(非主键字段)
  - 填写默认值时清空引用序列
  - 填写引用序列时清空默认值
- **关联矩阵 ⟷ 关联字典**
  - 选择关联矩阵时清空关联字典
  - 输入关联字典时清空关联矩阵

**主键字段特殊处理**:
- 主键字段不显示"默认值"输入框
- 只显示"引用序列"输入框
- 提供"设为自增"快捷按钮,一键设置为`AUTO_INCREMENT`

**设为NULL联动**:
- 点击"设为NULL"按钮时:
  - 默认值自动填入"null"
  - "是否可空"自动勾选为"是"
  - 引用序列自动清空
- "是否可空"未勾选时,"设为NULL"按钮禁用

### 3. 表状态可视化

**四种状态**:
- **未创建**(灰色):表结构配置完成,但未创建物理表
  - 显示蓝色创建图标
  - 没有字段时图标置灰不可点击
- **已创建**(蓝色):物理表已创建,配置与表结构一致
  - 隐藏创建和同步按钮
- **已同步**(绿色):字段配置修改后成功同步到物理表
  - 隐藏创建和同步按钮
- **待同步**(橙色):字段配置修改后,需要同步到物理表
  - 显示红色同步图标,提示立即同步

**按钮显示逻辑**:
- 创建图标和同步图标互斥显示
- 未创建状态:显示创建图标
- 待同步状态:显示同步图标
- 已创建/已同步状态:隐藏所有操作图标

**智能提示**:
- 没有字段时创建图标禁用,鼠标悬停提示"请先配置字段"
- 没有主键时弹出确认框,提示用户是否继续
- 待同步时红色图标醒目提示

### 4. 主键检查机制

**创建表前检查**:
```javascript
// 检查是否有字段标记为主键
const hasPrimaryKey = columns.some(col => col.isPrimaryKey === '1')

if (!hasPrimaryKey) {
  // 弹出确认框
  Modal.confirm({
    title: '确认创建表',
    content: '当前字段列表中没有主键字段,是否继续创建表?',
    onOk: () => executeCreateTable()
  })
}
```

**同步表前检查**:
- 同样的检查逻辑
- 确保用户知晓没有主键的风险

**建议**:
- 推荐为每个矩阵配置至少一个主键字段
- 主键通常使用BIGINT类型+AUTO_INCREMENT
- 主键字段名建议为`id`

### 5. 变更历史追溯

**记录的变更类型**:
1. 创建表(蓝色)
2. 添加字段(绿色)
3. 调整字段顺序(橙色)
4. 添加索引(青色)
5. 删除索引(红色)
6. 删除字段(品红色)
7. 修改表注释(紫色)

**展示信息**:
- **版本号**:每次变更递增
- **变更类型**:通过颜色和文字标识
- **影响字段**:显示相关字段名
- **DDL语句**:可展开查看具体执行的SQL
- **执行状态**:成功(绿色)或失败(红色)
- **错误信息**:失败时显示详细错误

**时间线布局**:
- 按版本分组,最新版本在上
- 同一版本的多个变更归为一组
- 时间从新到旧排列

### 6. 表单配置与布局

**自动字段关联**:
- 选择关联字段时,自动填充显示标签(来自字段注释)
- 自动设置合适的字段类型

**表单布局设计器**:
- 可视化拖拽调整字段位置
- 支持调整字段宽度和高度
- 60px网格系统+12px间隙
- 实时保存布局变更

**布局设计特点**:
- 栅格化布局,整齐对齐
- 支持字段跨列显示
- 自动避让,防止重叠
- 响应式设计

### 7. 联动配置

**触发事件**:
- 值改变(change)
- 失去焦点(blur)
- 获得焦点(focus)

**脚本支持**:
- 条件脚本:判断是否执行联动
- 动作脚本:执行的JavaScript代码
- 可访问表单中所有字段(通过字段名)

**联动示例**:

*自动计算总分*:
```javascript
// 条件脚本
score1 != null && score2 != null

// 动作脚本  
total_score = parseFloat(score1) + parseFloat(score2);
```

*根据年龄分组*:
```javascript
// 条件脚本
age != null

// 动作脚本
if (age < 18) {
  age_group = '未成年';
} else if (age < 60) {
  age_group = '成年人';
} else {
  age_group = '老年人';
}
```

### 8. 抽屉关闭自动刷新

**实现逻辑**:
```vue
<!-- ColumnConfig抽屉 -->
<a-drawer
  v-model:open="columnDrawerVisible"
  @close="handleColumnDrawerClose"
>
  <ColumnConfig :matrix="currentMatrix" />
</a-drawer>

// 关闭时刷新
const handleColumnDrawerClose = () => {
  loadData() // 重新加载矩阵列表
}
```

**好处**:
- 配置字段后,主列表立即显示最新状态
- 确保"创建表"按钮的启用/禁用状态正确
- 用户体验更流畅

### 9. 表注释双击编辑

**交互方式**:
- 双击表注释单元格进入编辑模式
- 输入框自动聚焦
- 失去焦点或按Enter保存
- ESC取消编辑

**实时保存**:
- 保存成功立即更新列表
- 保存失败自动恢复原值

## 应用场景

### 场景1:快速原型开发

无需编写代码,通过配置快速创建业务表和表单,加速原型开发。

**示例**:客户信息管理
- 配置客户信息表(姓名、电话、地址等)
- 配置表单(输入框、下拉选择等)
- 配置联动(选择城市自动填充区县)
- 立即可用,无需等待开发

### 场景2:动态业务对象

业务需求频繁变化时,通过配置快速调整表结构和表单,无需修改代码。

**示例**:调查问卷系统
- 根据不同调查主题动态创建表
- 灵活调整问题字段
- 修改表单布局
- 调整字段间联动关系

### 场景3:数据采集表

为不同的数据采集场景创建专用表,支持自定义字段和表单。

**示例**:设备巡检记录
- 不同设备类型有不同的巡检项
- 动态创建巡检记录表
- 配置专用表单
- 设置检查项之间的联动

### 场景4:配置化开发

减少重复的CRUD代码,通过配置实现常见的数据管理功能。

**示例**:产品档案管理
- 配置产品表(名称、分类、规格等)
- 自动生成CRUD界面
- 支持数据导入导出
- 无需编写代码

## 最佳实践

### 表设计

1. **合理规划字段**:创建前充分考虑业务需求
2. **必设主键**:每个表都应该有主键字段
3. **适当索引**:为常用查询字段创建索引
4. **字段命名**:使用有意义的英文名称,遵循小写+下划线规范

### 字段配置

1. **利用中文转拼音**:输入中文注释自动生成字段名
2. **正确使用互斥字段**:理解名称/顺序/父节点的含义
3. **合理设置默认值**:常用值设置默认,提升填写效率
4. **关联配置**:使用关联矩阵或字典简化数据录入

### 表单设计

1. **合理布局**:重要字段放在上方,相关字段就近放置
2. **字段提示**:添加描述信息,指导用户填写
3. **必填标识**:正确设置必填字段
4. **验证规则**:使用正则表达式验证输入格式

### 联动规则

1. **简单有效**:保持脚本简洁,避免过于复杂的逻辑
2. **条件明确**:条件脚本清晰判断是否执行
3. **容错处理**:考虑字段为空的情况
4. **性能优化**:避免在联动中执行耗时操作

### 运维管理

1. **变更历史**:定期查看变更历史,了解表结构演变
2. **及时同步**:字段修改后及时同步到物理表
3. **数据备份**:重要表定期备份数据
4. **清理无用表**:删除不再使用的矩阵配置

## 常见问题

### Q:表名前缀为什么不能修改?

A:设计考虑:
- 统一业务表命名规范
- 与系统表区分(sys_、t_等)
- 防止误操作导致表名混乱
- 如确需其他前缀,可联系管理员配置

### Q:字段名自动生成的拼音不准确怎么办?

A:解决方案:
- 自动生成仅作为初始建议
- 生成后可以手动修改
- 建议输入英文注释,更准确
- 或先输入字段名,再输入注释

### Q:为什么没有主键也能创建表?

A:设计考虑:
- 某些场景确实不需要主键(如日志表)
- 系统会弹出确认框,提醒用户
- 但强烈建议为每个表配置主键
- 主键有助于数据的唯一标识和性能优化

### Q:同步表结构会丢失数据吗?

A:安全说明:
- 同步表结构只修改表结构,不影响已有数据
- 但以下操作需谨慎:
  - 删除字段:会永久删除该列数据
  - 修改字段类型:可能导致数据转换失败
- 建议:重要操作前先备份数据

### Q:联动规则不生效怎么办?

A:排查步骤:
1. 检查联动规则是否启用
2. 使用"测试"功能验证脚本逻辑
3. 检查条件脚本是否返回true
4. 检查字段名是否正确
5. 查看浏览器控制台是否有错误

### Q:如何批量导入字段配置?

A:当前版本暂不支持,规划中的功能:
- Excel模板导入
- JSON配置导入
- 从其他矩阵复制字段
