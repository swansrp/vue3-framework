# 动态矩阵配置 - 快速开始

## 路由配置

### 方式1：通过后台管理配置（推荐）

在系统的菜单管理中添加：

```
菜单名称: 动态矩阵配置
路由路径: forge
组件路径: @/views/forge/index.vue
图标: TableOutlined
排序: 200
```

### 方式2：静态路由配置

在 `src/framework/router/index.ts` 中添加路由：

```typescript
import Forge from '@/views/forge/index.vue'

// 在路由配置中添加
{
  path: '/forge',
  name: 'Forge',
  component: Forge,
  meta: {
    title: '动态矩阵配置',
    icon: 'TableOutlined',
    keepAlive: true
  }
}
```

## 快速体验

### 示例1：创建用户评估表

#### 1. 创建矩阵

```
表名: user_score (系统会自动添加 biz_ 前缀)
表注释: 用户评估得分表
主键: id
引擎: InnoDB
字符集: utf8mb4
```

> ✨ **注意**: 表名前缀 `biz_` 为不可编辑部分，用户只需输入 `user_score`，系统保存为 `biz_user_score`

#### 2. 添加字段

| 字段名 | 注释 | 数据库类型 | 表单类型 | 必填 | 备注 |
|--------|------|-----------|---------|------|------|
| id | 主键 | BIGINT | - | - | 勾选“是否主键”，引用序列设为AUTO_INCREMENT |
| user_name | 用户姓名 | VARCHAR(50) | 单行文本 | 是 | 输入“用户姓名”后失去焦点，自动生成字段名 |
| score1 | 分数1 | DECIMAL(10,2) | 数字 | 是 | - |
| score2 | 分数2 | DECIMAL(10,2) | 数字 | 是 | - |
| total_score | 总分 | DECIMAL(10,2) | 数字 | 否 | 通过联动规则自动计算 |

> ✨ **智能功能**：
> - **中文转拼音**: 输入字段注释后，系统自动将中文转换为拼音生成字段名（仅新增时）
> - **互斥字段**: 
>   - 默认值 ↔ 引用序列（非主键时）
>   - 名称/顺序/父节点字段三选一
>   - 关联矩阵 ↔ 关联字典
> - **设为NULL按钮联动**: 点击后自动勾选“是否可空”

#### 3. 创建物理表

点击状态列的蓝色创建图标或“创建表”按钮，在数据库中创建真实的表。

> ✨ **提示**：
> - 如果没有配置主键字段，系统会弹出确认框
> - 建议至少配置一个主键字段（通常为id，设为自增）

#### 4. 配置表单

为每个字段创建表单配置：

**用户姓名：**
```
显示标签: 用户姓名
字段类型: 单行文本
必填: 是
验证规则: ^[\u4e00-\u9fa5]{2,10}$
```

**分数1：**
```
显示标签: 分数1
字段类型: 数字
必填: 是
最小值: 0
最大值: 100
```

**分数2：**
```
显示标签: 分数2
字段类型: 数字
必填: 是
最小值: 0
最大值: 100
```

**总分：**
```
显示标签: 总分
字段类型: 数字
只读: 是
```

#### 5. 配置联动

```
触发字段: 分数1
联动名称: 自动计算总分
触发事件: change
条件脚本: score1 != null && score2 != null
动作脚本: total_score = parseFloat(score1) + parseFloat(score2);
目标字段: total_score
```

#### 6. 填写数据

1. 切换到"数据管理"标签页
2. 点击"新增数据"
3. 填写用户姓名和两个分数
4. 总分会自动计算
5. 提交保存

## 常用场景

### 场景1：动态表单

适用于需要频繁调整表单字段的场景，如：
- 调查问卷
- 数据采集表
- 评估打分表

### 场景2：自定义业务对象

快速创建业务对象，无需修改代码：
- 客户信息管理
- 产品档案管理
- 订单信息管理

### 场景3：配置化开发

减少重复代码，提高开发效率：
- 通过配置实现CRUD
- 自动生成表单界面
- 灵活调整字段属性
