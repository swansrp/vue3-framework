# Dataset配置管理 - 实现说明

## 项目概述

本项目实现了一个完整的Dataset配置管理系统，允许用户通过SQL语句创建动态查询配置，无需编写代码即可实现复杂的多表查询和数据展示。

## 核心设计理念

### 1. SQL驱动配置
- 用户只需编写标准的SELECT SQL
- 系统自动解析SQL生成配置
- 支持多表JOIN和聚合函数

### 2. 低代码理念
- 配置即代码
- 可视化管理
- 快速迭代

### 3. 模块化设计
- 组件职责单一
- 高内聚低耦合
- 易于维护和扩展

**SQL生成逻辑**:
```typescript
// 构建SELECT子句
const selectPart = columns
  .filter(col => col.isVisible !== 'N')
  .map(col => `${col.columnSql} AS ${col.columnAlias}`)
  .join(',\n  ')

// 构建FROM子句
const fromPart = `FROM ${mainTable.datasetSql} ${mainTable.datasetAlias}`

// 构建JOIN子句
const joinParts = datasets
  .slice(1)
  .map(table => 
    `${table.joinType} JOIN ${table.datasetSql} ${table.datasetAlias} ON ${table.joinCondition}`
  )
  .join('\n')

// 组合完整SQL
const sql = `SELECT\n  ${selectPart}\n${fromPart}\n${joinParts}`
```

## 数据流设计

### 1. Dataset创建流程

```
用户输入SQL
    ↓
点击"预览解析结果"
    ↓
调用 /parse API
    ↓
展示解析结果（表配置、列配置）
    ↓
用户确认
    ↓
点击"确定"
    ↓
调用 /replace API
    ↓
保存配置
    ↓
刷新Dataset列表
```

### 2. 列配置修改流程

```
打开列配置
    ↓
调用 /column/list API 加载列表
    ↓
用户修改列信息
    ↓
调用 /column/update API
    ↓
刷新列列表
    ↓
触发父组件刷新Dataset列表
```

### 3. 数据查询流程

```
选择Dataset
    ↓
切换到"数据查询"标签
    ↓
根据列配置生成查询表单
    ↓
用户填写查询条件
    ↓
点击"查询"按钮
    ↓
构建查询条件（区分聚合/非聚合）
    ↓
调用 /data/query API
    ↓
展示查询结果
```

### 核心类型定义

```typescript
// Dataset配置信息
interface DatasetInfo {
  tableId: string              // Dataset ID
  dataSource?: string          // 数据源
  datasets?: DatasetTableInfo[] // 表配置
  columns?: DatasetColumnInfo[] // 列配置
}

// Dataset表信息
interface DatasetTableInfo {
  id?: string
  tableId: string
  datasetOrder: number         // 表顺序
  datasetSql: string           // 表名
  datasetAlias: string         // 表别名
  joinType?: string            // JOIN类型
  joinCondition?: string       // JOIN条件
  remark?: string              // 备注
}

// Dataset列配置信息
interface DatasetColumnInfo {
  id?: string
  tableId: string
  columnSql: string            // SQL表达式
  columnAlias: string          // 字段别名
  isAggregate?: string         // 是否聚合字段
  displayOrder?: number        // 显示顺序
  isVisible?: string           // 是否显示
  remark?: string              // 备注
}
```

## 总结

Dataset配置管理系统是一个典型的低代码配置系统，通过SQL驱动的方式实现了灵活的数据查询配置。整个系统采用模块化设计，组件职责清晰，易于维护和扩展。

**核心优势**:
1. **低代码**: 无需编写代码，通过SQL即可配置
2. **灵活性**: 支持复杂的多表JOIN和聚合查询
3. **易用性**: 可视化配置界面，操作直观
4. **可维护**: 模块化设计，代码结构清晰

**适用场景**:
- 动态报表配置
- 数据查询平台
- BI系统数据源配置
- 低代码平台数据层
