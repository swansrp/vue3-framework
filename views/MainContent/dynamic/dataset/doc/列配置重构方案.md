# 列配置功能重构方案

## 当前问题

列配置组件（ColumnConfig.vue）目前使用了单独的列操作接口：
- `datasetConfigAddColumn` - 添加列
- `datasetConfigUpdateColumn` - 更新列
- `deleteColumn` - 删除列
- `datasetConfigUpdateColumnsOrder` - 更新顺序

根据新的架构要求，所有列配置的修改应该统一使用 `parseSqlAndSave` (`/dataset/config/replace`) 接口。

## 重构目标

通过 SQL 重新解析和保存的方式来实现列配置的增删改，确保配置的一致性。

## 实现方案

### 方案一：基于 SQL 重构（推荐）

**核心思路：**
1. 列配置界面维护完整的 SQL 语句
2. 任何列的修改都通过重建 SQL 实现
3. 调用 `parseSqlAndSave` 替换整个配置

**优点：**
- 完全依赖 SQL 解析，配置一致性高
- 符合架构设计，统一使用 replace 接口
- 避免手动维护列配置与 SQL 不一致的问题

**缺点：**
- 需要实现 SQL 构建逻辑（从列配置 → SQL）
- 用户体验可能稍差（每次修改都需要重新解析）

**实现步骤：**

1. **添加 SQL 状态维护**
```typescript
const sqlState = ref<string>('')  // 维护当前 SQL

// 从 dataset 初始化 SQL
const initSql = () => {
  sqlState.value = buildSqlFromDataset(props.dataset)
}
```

2. **添加列时：修改 SQL**
```typescript
const handleAdd = async (newColumn) => {
  // 在 SQL 的 SELECT 部分添加新列
  const selectPart = extractSelectPart(sqlState.value)
  const newColumnSql = `${newColumn.columnSql} AS ${newColumn.columnAlias}`
  
  const updatedSql = insertColumnToSql(sqlState.value, newColumnSql)
  
  // 调用 parseSqlAndSave
  await parseSqlAndSave({
    datasetId: props.dataset.id,
    sql: updatedSql,
    dataSource: props.dataset.dataSource
  }, true, false)
  
  emit('refresh')
}
```

3. **编辑列时：修改 SQL**
```typescript
const handleEdit = async (oldColumn, newColumn) => {
  // 在 SQL 中找到并替换列定义
  const updatedSql = replaceColumnInSql(
    sqlState.value, 
    oldColumn.columnSql, 
    `${newColumn.columnSql} AS ${newColumn.columnAlias}`
  )
  
  await parseSqlAndSave({
    datasetId: props.dataset.id,
    sql: updatedSql,
    dataSource: props.dataset.dataSource
  }, true, false)
  
  emit('refresh')
}
```

4. **删除列时：修改 SQL**
```typescript
const handleDelete = async (column) => {
  // 从 SQL 中移除列
  const updatedSql = removeColumnFromSql(sqlState.value, column.columnSql)
  
  await parseSqlAndSave({
    datasetId: props.dataset.id,
    sql: updatedSql,
    dataSource: props.dataset.dataSource
  }, true, false)
  
  emit('refresh')
}
```

5. **调整顺序时：重建 SQL**
```typescript
const handleMoveUp = async (index) => {
  // 交换列在数组中的位置
  [dataSource.value[index], dataSource.value[index - 1]] = 
  [dataSource.value[index - 1], dataSource.value[index]]
  
  // 根据新顺序重建 SQL
  const updatedSql = rebuildSqlFromColumns(dataSource.value, props.dataset.datasets)
  
  await parseSqlAndSave({
    datasetId: props.dataset.id,
    sql: updatedSql,
    dataSource: props.dataset.dataSource
  }, true, false)
  
  emit('refresh')
}
```

### 方案二：保持现有接口（不推荐）

如果后端确实需要保留单独的列操作接口，前端继续使用现有接口即可。

但这与"列配置也应该使用 parseSqlAndSave"的要求不符。

## 需要新增的工具函数

```typescript
// 从 dataset 构建完整 SQL
function buildSqlFromDataset(dataset: DatasetInfo): string

// 向 SQL 插入新列
function insertColumnToSql(sql: string, columnSql: string): string

// 在 SQL 中替换列
function replaceColumnInSql(sql: string, oldColumn: string, newColumn: string): string

// 从 SQL 中移除列
function removeColumnFromSql(sql: string, columnSql: string): string

// 根据列数组重建 SQL
function rebuildSqlFromColumns(columns: DatasetColumnInfo[], tables: DatasetTableInfo[]): string

// 提取 SQL 的 SELECT 部分
function extractSelectPart(sql: string): string
```

## 建议

**推荐使用方案一**，原因：
1. 符合架构设计原则（统一使用 parseSqlAndSave）
2. 保证配置一致性（SQL 是唯一数据源）
3. 简化后端逻辑（不需要维护多个列操作接口）

**需要注意：**
- SQL 解析和构建逻辑需要健壮
- 提供友好的错误提示
- 考虑性能优化（避免频繁解析）
