# Portal配置快速使用指南

## 快速开始

### 1. Dataset的Portal配置

#### 步骤1: 创建Dataset
在"Dataset管理"Tab中创建或选择一个Dataset。

#### 步骤2: 生成Portal配置
1. 切换到"Portal配置"Tab
2. 点击左下角的"生成Portal配置"按钮
3. 填写表单：
   - **Portal名称**: 英文命名，建议格式 `ds_业务名称`（如：`ds_user_list`）
   - **显示名称**: 中文名称（如：`用户数据列表`）
4. 点击"确定"

#### 步骤3: 配置Portal
生成后会在左侧列表中看到新的Portal配置，点击即可编辑：
- **表格整体配置**: 分栏、弹框宽度、排序字段等
- **字段配置**: 显示、筛选、排序、编辑权限等

#### 步骤4: 使用Portal
在"数据查询"Tab或其他页面中使用Portal组件：

```vue
<portal
  table-id="ds_user_list"
  data-mode="DATASET"
  :reference-id="currentDataset.id"
/>
```

### 2. Matrix的Portal配置

#### 步骤1: 创建Matrix
在"矩阵管理"Tab中创建或选择一个Matrix，确保状态为"已生成"。

#### 步骤2: 生成Portal配置
1. 切换到"Portal配置"Tab
2. 点击左下角的"生成Portal配置"按钮
3. 填写表单：
   - **Portal名称**: 英文命名，建议格式 `mx_业务名称`（如：`mx_product_manage`）
   - **显示名称**: 中文名称（如：`产品管理表`）
4. 点击"确定"

#### 步骤3: 配置Portal
同Dataset配置流程。

#### 步骤4: 使用Portal
在"数据管理"Tab或其他页面中使用Portal组件：

```vue
<portal
  table-id="mx_product_manage"
  data-mode="MATRIX"
  :reference-id="currentMatrix.id"
/>
```

## 常见配置示例

### 示例1: 配置只读表格（仅查看）

1. 选择Portal配置
2. 在"表格整体配置"中：
   - 设置"只读"为开启
   - 设置"支持导入"为关闭
   - 设置"支持导出"为开启
3. 在"字段配置"中：
   - 所有字段的"新增显示"设为关闭
   - 所有字段的"编辑显示"设为关闭

### 示例2: 配置带筛选和排序的表格

1. 选择需要筛选的字段
2. 设置"是否筛选"为开启
3. 设置"是否排序"为开启
4. 在"表格整体配置"中设置默认排序字段

### 示例3: 配置树形表格

1. 在"表格整体配置"中：
   - 设置"id字段"（主键字段）
   - 设置"名称字段"（显示名称）
   - 设置"顺序字段"（排序字段）
   - 设置"父id字段"（父节点字段）
2. 开启"树形拖拽"（如需要支持拖拽调整层级）

## 多Portal配置管理

一个Matrix或Dataset可以有多个Portal配置，适用于不同场景：

### 示例: 用户管理的多Portal配置

#### Portal 1: `ds_user_list` - 用户列表
- **用途**: 展示用户基本信息
- **字段**: 用户名、姓名、部门、状态
- **特点**: 只读、支持筛选和排序

#### Portal 2: `ds_user_manage` - 用户管理
- **用途**: 完整的用户CRUD操作
- **字段**: 所有用户字段
- **特点**: 可增删改、详细表单

#### Portal 3: `ds_user_mobile` - 移动端用户
- **用途**: 移动端展示
- **字段**: 精简字段
- **特点**: 卡片式显示、简化操作

## 角色权限配置

Portal支持按角色配置不同的显示方式：

### 步骤1: 选择角色
在顶部下拉框选择角色（默认为"默认配置"）

### 步骤2: 初始化角色配置
如果该角色没有配置，点击"初始化"按钮，选择一个模板角色

### 步骤3: 配置差异化
为不同角色配置不同的：
- 字段可见性
- 操作权限
- 默认筛选条件

## 配置导入导出

### 导出配置
1. 选择要导出的Portal配置
2. 点击右上角"导出配置"按钮
3. 系统下载 `.dat` 文件

### 导入配置
1. 选择目标Portal配置
2. 点击"导入配置"按钮
3. 选择之前导出的 `.dat` 文件
4. 确认导入

**注意**: 导入会覆盖当前配置！

## 高级功能

### 1. 默认筛选条件

为Portal设置默认的筛选条件：

1. 在"表格整体配置"中点击"默认筛选条件"按钮
2. 配置筛选规则（如：状态=启用，部门=IT部）
3. 保存后，每次打开该Portal都会自动应用这些筛选

### 2. 关联配置

为Portal配置关联项（如关联其他表格的弹窗选择）：

1. 点击"关联配置"按钮
2. 在弹窗中配置关联关系
3. 保存

### 3. 统计指标

为Portal添加统计指标卡片：

1. 点击"统计指标"按钮
2. 配置统计字段和计算方式
3. 指标会显示在表格上方

### 4. 通用图表

为Portal添加可视化图表：

1. 点击"通用图表"按钮
2. 配置图表类型（柱状图、饼图、折线图等）
3. 图表会显示在独立页面

## 常见问题

### Q1: Portal配置Tab为什么是禁用状态？
**A**: 需要先在管理Tab中选择一个Matrix或Dataset。

### Q2: 生成Portal配置后找不到？
**A**: 检查角色选择是否正确，默认应该选择"默认配置"。

### Q3: 能否复制Portal配置？
**A**: 可以！右键点击Portal配置，选择"复制"，输入新的Portal名称即可。

### Q4: 修改配置后需要刷新页面吗？
**A**: 不需要，配置实时生效。建议使用 `Ctrl+S` 快捷键保存。

### Q5: Portal名称可以修改吗？
**A**: Portal名称（tableId）生成后不可修改，但显示名称可以随时修改。

## 最佳实践

### 1. 命名规范
- Dataset Portal: `ds_模块_功能`（如：`ds_user_list`）
- Matrix Portal: `mx_模块_功能`（如：`mx_product_manage`）
- 显示名称使用中文，清晰描述用途

### 2. 字段配置
- 主键字段通常不显示在表格中
- 时间字段建议设置排序功能
- 状态字段建议设置筛选功能
- 操作密集的表格减少字段数量

### 3. 性能优化
- 大数据量表格不要开启过多筛选字段
- 合理设置分页大小
- 避免在移动端显示过多列

### 4. 权限设计
- 为不同角色创建专属Portal配置
- 敏感字段仅在特定角色配置中显示
- 使用"只读"模式限制操作权限
