# 字段排序功能优化说明

## 优化背景

### 原有问题
1. **拖动不便**: 字段列表中字段较多时，拖动调整顺序操作困难
2. **筛选冲突**: 筛选后拖动会导致顺序错乱（因为只能看到部分字段）
3. **体验不佳**: 无法一边筛选查看一边调整完整的字段顺序

### 优化方案
创建一个专门的**字段排序管理弹窗**，实现：
- 显示所有字段，支持拖拽排序
- 筛选条件仅影响字段高亮显示（灰化非匹配字段）
- 拖动时始终操作完整的字段列表

## 功能特性

### 1. 统一操作入口
在字段列表表头右侧，有一个控制图标（⚙️），点击后弹出菜单，包含：
- **排序操作**: 字段排序管理
- **筛选条件**: 筛选不同类型的字段

### 2. 菜单界面

#### 菜单布局（新设计）
```
┌─────────────────────────┐
│ 字段列表            [⚙️] │ ← 点击齿轮图标
└─────────────────────────┘

弹出菜单:
┌─────────────────────────┐
│ 排序操作                │
│ [🔽 字段排序管理]       │
│ ─────────────────────   │
│ 筛选条件                │
│ [选择筛选类型 ▼]        │
│ [显示全部]              │
└─────────────────────────┘
```
- **全部字段**: 显示所有字段（默认）
- **有效字段**: 只高亮 enable=1 的字段
- **表格显示**: 只高亮 show=1 的字段
- **详情显示**: 只高亮 detailShow=1 的字段
- **新增显示**: 只高亮 addShow=1 的字段
- **编辑显示**: 只高亮 editShow=1 的字段

#### 字段列表（中间）
每个字段项显示：
```
[拖动图标] 字段显示名称                 [有效][表格][详情] [顺序号]
           字段属性名(property)
```

**状态标签颜色：**
- 有效: 绿色
- 表格: 蓝色
- 详情: 青色
- 新增: 紫色
- 编辑: 橙色

**高亮/灰化效果：**
- 匹配筛选条件的字段：正常显示（高亮）
- 不匹配的字段：灰化显示（opacity: 0.4）
- 拖动过程中的字段：半透明（opacity: 0.5）

#### 操作按钮（底部）
- **提示**: "拖动字段可调整顺序，筛选条件仅影响高亮显示"
- **取消**: 关闭弹窗，不保存更改
- **确定**: 保存排序更改并刷新列表

### 3. 拖拽交互

#### 拖拽方式
- 整个字段项都可拖拽（cursor: move）
- 拖拽手柄图标提示可拖动

#### 拖拽过程
1. 开始拖动：字段项变为半透明
2. 拖动中：经过其他字段时自动重新排列
3. 放下：立即更新顺序号
4. 结束：恢复正常显示

#### 拖拽特点
- ✅ **所有字段都参与排序**（包括灰化的字段）
- ✅ **筛选不影响拖拽范围**（只影响视觉高亮）
- ✅ **实时更新顺序号**（右侧数字实时变化）

## 技术实现

### 核心文件
- **组件文件**: `src/framework/views/MainContent/PortalConfig/components/ColumnOrderModal.vue`
- **主页面**: `src/framework/views/MainContent/PortalConfig/index.vue`

### 关键技术点

#### 1. 原生拖拽API
使用HTML5原生拖拽API，无需额外依赖：
```typescript
// 拖拽事件
@dragstart="handleDragStart($event, index)"
@dragover="handleDragOver($event, index)"
@dragend="handleDragEnd"
@drop="handleDrop($event, index)"
```

#### 2. 筛选高亮逻辑
```typescript
const isHighlighted = (column: ColumnItem): boolean => {
  if (filterType.value === 'all') {
    return true  // 全部字段都高亮
  }
  
  if (filterType.value === 'enable') {
    return column.enable === '1'
  }
  
  return column[filterType.value] === '1'
}
```

#### 3. 拖拽排序逻辑
```typescript
const handleDrop = (e: DragEvent, dropIndex: number) => {
  const draggedItem = localColumns.value[draggedIndex.value]
  const newColumns = [...localColumns.value]
  
  // 移除拖动的元素
  newColumns.splice(draggedIndex.value, 1)
  
  // 插入到新位置
  newColumns.splice(dropIndex, 0, draggedItem)
  
  localColumns.value = newColumns
  updateDisplayOrder()  // 更新displayOrder字段
}
```

#### 4. 数据同步
确定按钮点击后：
```typescript
const handleColumnOrderConfirm = (orderedColumns: any[]) => {
  // 更新本地数据
  fieldRecords.value = orderedColumns
  tableConfig.value.columns = orderedColumns
  
  // 提交到后端
  const columnOrder = orderedColumns.map((column, index) => ({
    id: column.id,
    title: column.displayName,
    showOrder: index + 1
  }))
  
  updatePortalColumnOrder(columnOrder).then(() => {
    onSearch()  // 刷新列表
  })
}
```

## 使用流程

### 场景1: 调整所有字段顺序
1. 点击字段列表表头的排序图标
2. 选择"全部字段"筛选
3. 拖动字段调整顺序
4. 点击"确定"保存

### 场景2: 只关注表格显示字段的顺序
1. 点击字段列表表头的排序图标
2. 选择"表格显示"筛选
3. 高亮的字段就是表格中显示的字段
4. 拖动调整这些字段的顺序（灰色字段也会一起移动）
5. 点击"确定"保存

### 场景3: 调整有效字段到顶部
1. 点击排序图标
2. 选择"有效字段"筛选
3. 将高亮的有效字段拖到列表顶部
4. 点击"确定"保存

## 对比优化前后

| 特性 | 优化前 | 优化后 |
|------|--------|--------|
| **拖动范围** | 直接在列表中拖动 | 在弹窗中拖动 |
| **可见范围** | 受列表高度限制 | 最大600px，可滚动 |
| **筛选影响** | 筛选后只能拖动可见字段 | 筛选只影响高亮，所有字段可拖动 |
| **顺序准确性** | 筛选后容易错乱 | 始终保持完整顺序 |
| **操作便利性** | 字段多时不便操作 | 清晰的弹窗界面 |
| **视觉反馈** | 无筛选状态指示 | 高亮/灰化明确区分 |
| **状态标签** | 无 | 显示字段的所有状态 |
| **实时更新** | 拖动后才更新 | 拖动过程实时更新 |

## 注意事项

1. **筛选仅高亮**: 筛选条件不会隐藏字段，只会灰化不匹配的字段
2. **全量排序**: 无论筛选什么条件，拖动排序始终影响所有字段
3. **顺序同步**: 确定后会立即同步到后端和主列表
4. **取消不保存**: 点击取消或关闭弹窗，所有更改会丢失
